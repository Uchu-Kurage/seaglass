# 機能仕様 — 留守中の打ち上げのお知らせ

## 目的

久しぶりに浜を訪れたとき、留守のあいだに何かが打ち上がっていたことを、そっと伝える。
「時々のぞきに来ると、新しい贈りものが届いている」という、この作品の時間のリズムを
プレイヤーに実感してもらうための、控えめな演出。

## 体験のイメージ

浜に入った瞬間、画面の中ほどに、短い一文が静かに現れて、数秒かけてゆっくり消える。

> 留守のあいだに、三つ 届いていました。

数を強調したり、通知バッジのように主張したりしない。潮騒のように、気づけばそこにある程度の
存在感にとどめる。効果音・振動・派手なアニメーションは使わない。

## 表示条件

- 浜に入ったときに一度だけ判定する。
- 前回この浜を離れてから **今回の入場までに新しく打ち上がった数**（= `advanceShore` が
  今回追加した個体数）が **1個以上** のときだけ表示する。0個なら何も出さない。
- はじめて訪れる浜（初回訪問で `FIRST_VISIT` 個が置かれる場合）では出さない。
  「留守のあいだに」は再訪の文脈でのみ意味を持つため。

## 文面

数に応じて和文で自然に。数字は漢数字が望ましいが、10以上は算用数字でよい。

- 1個: 「留守のあいだに、ひとつ 届いていました。」
- 2〜9個: 「留守のあいだに、〈漢数字〉つ 届いていました。」（三つ、四つ… の数え方）
- 10個以上: 「留守のあいだに、たくさん 届いていました。」

※ 厳密な数を出すより、少ない数は具体的に、多いときは「たくさん」と包む方が、この作品の
トーンに合う。実装が難しければ全て「〈数〉個 届いていました」でも許容するが、まず上記を試みること。

## 見た目

- 位置・字色・字送りは、既存の「潮が静かです」案内（`onShore === 0` のときの表示）と
  同じ設計言語にそろえる。中央やや上、明朝、淡い墨色、`letter-spacing` 広め、
  白い薄いテキストシャドウ。
- 表示時間は 4〜5 秒程度。フェードインしてしばらく留まり、フェードアウトして消える。
- 消えたあとは再表示しない（同じ入場中に再びは出さない）。
- 「砂浜のものに、触れてみましょう」のヒントと同時に出さない。お知らせを優先し、
  お知らせが消えてから通常のヒント判定に戻す、くらいの譲り合いが望ましい。

## 実装のヒント（あくまで参考。既存コードに合わせて調整可）

- 打ち上がった数は `advanceShore` が知っている。この関数が「今回新しく追加した数」を
  返せるようにするか、戻り値に含める形にすると、呼び出し側（渚の読み込み effect）で
  受け取れる。初回訪問（`items === null` の分岐）では 0 を返すこと。
- 受け取った数を `BeachScene` の state（例: `arrivals`）に入れ、1以上なら数秒後に
  自分で消えるお知らせ要素を表示する。`setTimeout` での自動消去で十分。
- 既存の後方互換や `storage` 両対応の仕組みは触らない。

## やらないこと（スコープ外）

- OS のプッシュ通知やタブのバッジなど、アプリ外への通知は一切しない。あくまで画面内の一文。
- 数の履歴を残す・累計を見せる等はしない。今回はその場限りのそっとした挨拶にとどめる。

## 完了の目安

- 数時間〜1日ほど間をあけて再訪すると、留守中に届いた数が静かに表示される。
- 打ち上がりが0のときや初回訪問では表示されない。
- 既存のセーブ・デプロイ・初期配置のいずれも壊れていない（`npm run build` が通る）。
