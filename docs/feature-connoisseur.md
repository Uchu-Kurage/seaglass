# 機能仕様：目利きと名品棚（厳選）

> 対象リポジトリ: `seaglass`
> 想定配置: `docs/feature-connoisseur.md`
> 実装環境: Claude Code（本ドキュメントを起点に PR を作成）
> 関連: `CLAUDE.md`（憲章）, `docs/feature-sea-rhythm.md`

---

## 0. 目的とトーン

「厳選」を、**リセット連打で理想個体を引く**遊びとしてではなく、**眼が肥え、揃ったものから選んで飾る**遊びとして実装する。この海の理屈に翻訳した厳選。

- **やらないこと**：ソフトリセット／引き直し／数値ゲージ／「あと◯個で」の煽り。個体の形・質は seed 由来で**決してリセットしない**。
- **やること**：個体の隠れた見どころが、集めるほど**見えるようになる**（＝目が肥える）。揃った中から数点を選び、特別な場所に**飾る**（＝厳選の所作）。
- 稀な一片との出会いは、狙い撃ちではなく**ゆっくり通う先に静かに訪れる**もの。フィードバックは極控えめ。

---

## 1. 見どころ（個体の質・seed 由来・不変）

各個体は既に seed を持つ。そこから**決定論的に**質を導く（seed が同じなら常に同じ。専用保存は不要、都度算出でよい）。**形と同じく、後から変化しない。**

既定の見どころ軸（4つ）：

| 軸 | 表示例（数値ではなく叙情語） |
|---|---|
| 色 | 既存の色。稀な色（赤・青）を含む |
| 透明度 | 「澄んでいる」「半ば透ける」「曇っている」 |
| かたち | 「まるい」「いびつ」「稜のある」／稀に「しずく」「まんまる」「ちいさな心臓」 |
| 磨き（曇りの質） | 「よく凪いだ」「ざらつきの残る」「霜のような」 |

- **数値は一切表に出さない。** 内部でスコア相当を持ってよいが、UI は叙情的な語のみ。
- 稀少の兆し（§4）は、これらの組み合わせ（稀な色 × 稀なかたち × 澄み 等）から導く。
- ※ 将来、`磨き` 軸だけは「海に還して磨く（育成）」で経時変化させる余地がある。**本仕様では固定**とし、育成は別機能として切り離す（フック名だけ意識しておく）。

### 1.1 叙情語（語彙案・オリジナル）

seed 由来の内部値を、次の語に写像する（数値は出さない）。各軸は seed から段に振り分ける。**すべて叩き台。差し替え可。**

**色**（既存の色に叙情名を与える。常に見える）
| 色 | 叙情名 | 希少 |
|---|---|---|
| 白・透明 | しろ／なぎさ色 | ふつう |
| 緑 | みどり／若葉の底 | ふつう |
| 水色 | みずいろ／浅瀬 | ふつう |
| 琥珀・茶 | こはく／夕暮れの澱 | ややまれ |
| 青 | あお／深みの色 | まれ |
| 赤 | あか／いちばん遠い色 | ごくまれ |

**透明度**（にごり → 澄み）
- 「霧をふくむ」→「すりガラスめく」→「半ば透ける」→「澄んでいる」→「水のように澄んで」

**かたち**（ふつう → 稀なかたち）
- ふつう：「角の残る」「平たく薄い」「まるみを帯びた」「ころんと丸い」
- 稀：「しずくのかたち」「まんまる」「ちいさな心臓」「勾玉のよう」

**磨き**（洗われたて → 深く凪いだ。※霜のような曇りが珍重される）
- 「艶の残る」→「ざらつきのある」→「なめらかな」→「霜をまとう」→「深く凪いだ」

**組み立てテンプレート**（眼のくらいが高いほど語が増える）：
`{磨き}、{透明度}{色}の、{かたち}`
例）「霜をまとう、澄んだ あおの、しずくのかたち」／くらい1では「あお」だけ。

## 2. 目が肥える（見え方の深化）

はじめは色くらいしか見えない。集めるほど**眼の位（くらい）**が上がり、詳細ビューで見える軸が増える。**力ではなく"知覚"が開いていく進行**。

```
目のくらい（eyeLevel）: 収集した個体総数から導出（保存しない）
  くらい1（初期）        : 色 のみ
  くらい2（総数 ≈ 20）   : ＋ 透明度
  くらい3（総数 ≈ 50）   : ＋ かたち
  くらい4（総数 ≈ 100）  : ＋ 磨き ＋ 稀少の気づき（§4）
```

- **単一の位（eyeLevel）で全軸をまとめて開放する（確定）。** 軸ごとに分けない。位が上がるたび、見える軸が一段ずつ増える。
- しきい値（20/50/100）は要調整。段数は現状4段。
- **過去に拾った個体にも遡って適用**される（質は最初からそこに在り、見えるようになるだけ）。昔の一片を見返して新しく気づく——この体験を大切にする。
- くらいが上がったことは、**そっと**知らせる程度（演出しすぎない）。ゲージや「レベルアップ！」は出さない。

## 3. 窓辺（飾る＝厳選の所作）

**枠の限られた展示場所**を用意する。持ち帰りの袋（§5）から数点を選んで飾る。**枠が少ないから、選ぶことに意味が生まれる。** これが厳選の本体。

- **場所の名＝「窓辺」（推奨・確定）。** 硝子は光の通り方で見える。窓辺なら、既存の time-of-day ライティングで**時刻ごとに光り方が変わる**——この作品にいちばん似合う置き場所。「名品棚」のようなトロフィー感のある語は避ける。
- **枠数：初期 3。** 収集の節目でゆっくり増える：**総数 60 / 150 / 300 で +1、上限 6（推奨）**。窓辺は親密な場所なので、数は小さく保つ。← しきい値は調整可。
- 所作：袋から個体を選んで空き枠に置く／入れ替える。**優劣の点数やおすすめは出さない**。何を名品とするかは**プレイヤーの眼が決める**。
- 表示：飾った個体は独立ビュー（§5）に静かに並ぶ。**時刻のライティング**で光り方が変わる。
- 飾っても収集データは変わらない（窓辺の一点は袋の個体への参照。袋からは消えない）。

## 4. 稀少の気づき（極控えめ）

- seed 由来の質から**稀少度**を導く（稀な色・稀なかたち・高い澄み、あるいはそれらの重なり）。
- **ひとことだけ静かに添える（確定）。** 稀な個体を**拾ったとき**、または**眼のくらいがその軸に届いたとき**、詳細ビューに一行そっと出す。**通知・光・音の派手さは無し。ランキングや「レア度★」は出さない。**
- 「昔ひろった一片が、眼が肥えてから稀少と気づく」遅れての発見も起こりうる。むしろ歓迎。

**一言の語彙案（オリジナル・叩き台）**：何がその一片を特別にしているかで、そっと言い換える（順位は言わない）。
- 赤：「これは、赤。この渚でいちばん遠い色」
- 青：「青は、めったに戻ってこない」
- 稀なかたち：「こんなかたちに出会うのは、久しぶりだ」
- 際立つ澄み：「これほど澄んだものは、そうない」
- 重なり（色×かたち等）：「これは、ちょっとした宝かもしれない」

## 5. 描画・UI（袋と窓辺）

**ライブラリの構成を、拾う→選ぶ の流れに合わせて整理する（確定）。**

- **持ち帰りの袋（旧「棚（全個体）」を改称）**：浜で拾ったすべての個体が入る場所。これまでの個体一覧タブをそのまま「持ち帰りの袋」にリネームする。データ構造は変えない（表示名と、あれば見出しのアイコン/文言のみ変更）。
- **種カタログ**：従来どおり（変更なし）。
- **窓辺（独立ビュー・新規）**：袋から選んで飾った数点だけを並べる、静かな独立画面。ライブラリのタブ内ではなく、**独立した面**として置く。空き枠は控えめな余白。time-of-day ライティングで光り方が変わる。入れ替えは既存 UI の作法（長押し／タップで差し替え等）に合わせる。
- **個体詳細ビュー**：袋（または窓辺）から一点を開くと、いつ・どの浜で拾ったか＋**眼のくらいに応じた見どころ**を叙情語で（§1.1 のテンプレート）表示。稀少の一言があればそっと添える。ここから「窓辺に飾る／窓辺からおろす」操作ができる。

> 用語整理：**袋＝持ち帰ったすべて**、**窓辺＝選んで飾る少数**。「棚」の語は使わない（袋と窓辺で役割が明確なため）。

## 6. 保存スキーマ（例）

```json
"curio": {
  "display": [
    { "seed": 1837465, "beachId": "<beachId>", "foundAt": "2026-07-12T08:20:00Z", "slot": 0 },
    { "seed": 9182003, "beachId": "<beachId>", "foundAt": "2026-07-15T07:05:00Z", "slot": 1 }
  ]
}
```

- 保存するのは**プレイヤーの選択（窓辺に飾った個体と並び）だけ**。`seed + beachId + foundAt` で袋の個体を参照する。
- 見どころ・稀少度・眼のくらいは**すべて既存データ＋seed から都度導出**（保存しない）。専用の重複データを持たない。
- 保存キーは `seaglass-curio`（`seaglass-` 接頭辞・抽象化レイヤ経由）。
- 後方互換：`curio` 未定義は空表示にフォールバック。

## 7. 既存システム・憲章との整合

- **形は決してリセットしない**：質も seed 由来で不変。眼が変わるのは見え方だけ。
- **極控えめ**：数値・ゲージ・レア度★・達成演出を出さない。叙情語と余白で示す。
- **スロータイム**：稀な個体は狙い撃ちできない。1時間1個の歩みの先に訪れる。
- **稀な色（赤・青）** は機能②（時化）・③（小瓶）と共有の希少色。厳選の"名品"候補として自然に連結する。
- **決定論・二重保存・後方互換・実在地名を出さない**：既存の約束をすべて踏襲。

## 8. 受け入れ基準

- [ ] 同一 seed の個体は、質・稀少度が常に同じ（リロードで揺れない）。
- [ ] 眼のくらい（単一）が上がると、過去の個体も含め見える見どころが一段ずつ増える。
- [ ] 見どころ・稀少はすべて叙情語で、数値やレア度★を表に出さない。
- [ ] 旧「棚（全個体）」が「持ち帰りの袋」に改称され、データは保たれる。
- [ ] 「窓辺」は独立ビューで、枠が限られ（初期3）、飾る／入れ替えができ、優劣の点数を出さない。
- [ ] 窓辺に飾っても袋の個体データは変わらない。
- [ ] `curio` は `seaglass-` 接頭辞・抽象化レイヤ経由で保存され、未定義でも壊れない。
